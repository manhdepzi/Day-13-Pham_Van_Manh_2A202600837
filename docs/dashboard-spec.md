# Dashboard Specification

Implementation: [`dashboard.html`](dashboard.html), a self-contained local HTML
dashboard generated by `python scripts/build_dashboard.py` from the JSON
snapshots under `data/evidence/`.

This lab does not deploy Prometheus or Grafana. The HTML artifact demonstrates
the required panels and incident comparisons without claiming production
time-series persistence.

Report settings:

- Refresh model: regenerate with `python scripts/build_dashboard.py`
- Source: exported evidence JSON
- Threshold and unit labels must remain visible

## 1. Request Latency

- Metrics: `p50_latency_ms`, `p95_latency_ms`, `p99_latency_ms`
- Unit: milliseconds
- Why: tail latency shows user-visible degradation hidden by averages.
- Normal: P95 around 150-200 ms in the local fake pipeline.
- Incident: `rag_slow` produces a request near 2650 ms and raises P95/P99.
- SLO line: 3000 ms.

## 2. Error Rate

- Metrics: `error_rate_pct`, `total_errors`, `error_breakdown`
- Unit: percent and request count
- Why: measures failed user requests and identifies the dominant error type.
- Normal: 0%.
- Incident: `tool_fail` creates `RuntimeError`; captured evidence reached 5.88%.
- Alert line: 5%.

## 3. RAG Latency

- Metrics: `rag_p50_latency_ms`, `rag_p95_latency_ms`
- Unit: milliseconds
- Why: separates retrieval latency from model latency.
- Normal: close to 0 ms for local retrieval.
- Incident: `rag_slow` adds approximately 2500 ms to `rag.retrieve`.
- Threshold line: 2000 ms.

## 4. LLM Token Usage

- Metrics: `total_input_tokens`, `total_output_tokens`,
  `average_input_tokens`, `average_output_tokens`
- Unit: tokens
- Why: token growth drives model cost and can expose prompt/output regressions.
- Normal: sample output is about 20-35 tokens.
- Incident: `cost_spike` multiplies output usage by four; the latest evidence
  request shows 20 to 80.
- Threshold line: two times the baseline average output tokens.

## 5. Estimated Cost

- Metrics: `total_cost`, `average_cost`
- Unit: USD
- Why: translates token behavior into budget impact.
- Normal: roughly USD 0.0004-0.0007 per local sample.
- Incident: the latest cost-spike request reached USD 0.001311.
- Budget line: USD 2.50 daily lab budget.

## 6. Quality Score

- Metrics: `average_quality_score`, `quality_pass_rate`
- Unit: score 0-1 and percent
- Why: latency and cost improvements are not useful if answer quality falls.
- Normal: known-domain requests should score at least 0.75.
- Incident: current incidents target reliability/performance, so quality should
  remain stable for successful responses.
- SLO line: average score 0.75; pass threshold 0.75.

## Evidence

Use the generated files in `data/evidence/` to demonstrate before/after
behavior. A production implementation would scrape Prometheus time series;
this lab intentionally uses in-memory snapshots.

The generated dashboard embeds the evidence values so it also opens directly
through `file://` without fetch/CORS or a running API.
